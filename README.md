## Lead Capture Setup (Formspree + Supabase Fallback)

The expression of interest form on `/events` posts to `/api/lead-interest`.

- Primary destination: Formspree (`https://formspree.io/f/xqeywzzq`)
- Fallback destination: Supabase table (if Formspree fails)
- Emergency fallback: local queue file at `/tmp/expedition-america-leads-backup.jsonl`

### 1) Configure environment variables

Add these to `.env.local`:

```bash
SUPABASE_URL="https://YOUR_PROJECT_ID.supabase.co"
SUPABASE_SERVICE_ROLE_KEY="YOUR_SUPABASE_SERVICE_ROLE_KEY"
SUPABASE_LEADS_TABLE="lead_submissions"
```

Restart the Next.js server after changing env values.

### 2) Create the fallback table in Supabase

Run this SQL in the Supabase SQL editor:

```sql
create table if not exists public.lead_submissions (
  id bigint generated by default as identity primary key,
  created_at timestamptz not null default now(),
  timestamp text,
  source text,
  name text,
  email text,
  city text,
  dates text,
  experience text,
  notes text,
  backup_reason text
);
```

If you want a different table name, set `SUPABASE_LEADS_TABLE` to match it.

## NYC Group Tour Email Setup

The form on `/nyc-group-tour` posts to `/api/nyc-group-tour`, which sends the email through EmailJS from the server.

Add these to `.env.local`:

```bash
EMAILJS_SERVICE_ID="YOUR_EMAILJS_SERVICE_ID"
EMAILJS_TEMPLATE_ID="YOUR_EMAILJS_TEMPLATE_ID"
EMAILJS_PUBLIC_KEY="YOUR_EMAILJS_PUBLIC_KEY"
EMAILJS_PRIVATE_KEY="YOUR_EMAILJS_PRIVATE_KEY"
```

Your EmailJS account must allow API access for non-browser applications, since the request is sent from the Next.js server route.
If EmailJS strict mode is enabled, the private key is required.
